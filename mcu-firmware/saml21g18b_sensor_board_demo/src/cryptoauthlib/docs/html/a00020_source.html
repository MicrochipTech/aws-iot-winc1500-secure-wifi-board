84 // this is the ATCACommand parameter structure. The caller to the command method must
85 // initialize param1, param2 and data if appropriate. The command method will fill in the rest
86 // and initialize the packet so it's ready to send via the ATCAIFace.
87 // this particular structure mimics the ATSHA and ATECC family device's command structures

89 // Note: pack @ 2 is required, @ 1 causes word alignment crash (though it should not), a known bug in GCC.
90 // @2, the wire still has the intended byte alignment with arm-eabi. this is likely the least portable part of atca

110  // and implementation
